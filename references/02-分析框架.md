# 02 · 分析框架（价值哲学 / 指标 / 诊断 / 建议）

> 这是分析师的"脑子"。规则全在 `scripts/analyze.py` 里，**透明、可调**。改阈值就改文件顶部常量。

## 0 · 价值哲学（判断 > 数据）

**用户不缺数据——后台一打开全是数字。缺的是判断。** 一个叫"分析师"的东西，价值不在"把数字画成图"（那是仪表盘，零增量），而在**降低用户下一个决策的不确定性**：这周该赌哪边、有多大把握、别在什么上浪费时间。

价值阶梯，越往上越值钱，钱在上三层：

```
描述：发生了什么   ← 后台就给，最不值钱（别停在这层）
诊断：为什么发生
预测：接下来会怎样
指示：那我该做什么 ← 最值钱，但最易沦为通用废话（必须挂数据）
```

**七个价值点**（按被低估程度排）：

1. **闭环验证**——上期建议到底有没有用？analyst 与"报告生成器"的分水岭。没闭环的建议都是放空炮。实现：`复盘历史.json` 存每期结论+指标，下期自动对比（上次任务 → 指标怎么动 → 有没有发新内容）。
2. **止损 / 聚焦**——结论只给**一件事**，并显式说"别做什么"。负向建议比正向更省用户时间；小号死于"什么都想优化"。
3. **复制标杆**——建议永远围绕观看最高那条（动态识别 `winner`）展开，把"偶然爆"变"可复制"。
4. **归因**——给"为什么"不只给"是什么"。
5. **基准**——三把尺子：自己的过去（环比）、同赛道（对标，待接 cookie）、目标（还差多少）。
6. **预警**——趋势方向比绝对值更早暴露问题。
7. **置信度诚实**——样本薄就标"方向性判断"，不假装确定。假装确定是骗术。

## 报告结构（判断优先 · build_report.py 已实现）

```
① 结论（英雄区）   账号状态一句话 + 本周唯一一件事 + 止损「别做」 + 置信度
② 上期复盘         闭环验证：上次任务 → 指标变化(↑↓) → 有没有执行
③ 为什么(诊断)+怎么做(建议)   判断层
──────── 支撑数据 ────────（分隔线）
④ KPI / 漏斗 / 趋势 / 红黑榜   降级为证据
```

> 翻转原则：把"非看数据不可知"的判断放最前，把后台自己就能看到的数字放最后。

## 指标体系

| 指标 | 定义 | 看什么 |
|---|---|---|
| 涨粉漏斗 | 观看 → 主页访问 → 涨粉（近7日） | 瓶颈在"看得到/点进来/留得住"哪一环 |
| 观看转主页率 | 主页访问 ÷ 观看 | 内容有没有引导读者点主页（系列感/钩子） |
| 主页转涨粉率 | 涨粉 ÷ 主页访问 | 主页（置顶/bio/封面统一度）留不留得住人 |
| 单篇出池 | 单篇观看 ≥ 阈值 | 有没有冲出冷启动池（小红书每篇先发 200–500 人试水） |
| 藏赞合计 | 赞 + 藏 | 内容质量代理；高藏赞+低观看 = 好内容没被看见 |
| 观看环比 | 本次快照 − 上次快照 | 哪条还在爬（放大/置顶），哪条已死 |

## 阈值（analyze.py 顶部常量，按账号体量调）

```python
COLD = 300            # 单篇观看 < 300 ≈ 没出冷启动池
LOW_HOME_RATE = 3.0   # 观看→主页 < 3% 偏低
LOW_FOLLOW_RATE = 8.0 # 主页→涨粉 < 8% 偏低
```

> 阈值随账号体量/赛道调整：粉丝量大、流量池大的账号 COLD 应上调；垂类小众可下调。

## 诊断逻辑（判断瓶颈在哪、别在哪儿白费力）

按"账号阶段"分两态，**出池(曝光) 永远是转化的前提**：

1. **均观看 < COLD（还没出池）** → 病灶＝没被看见。阶段任务只有一个：让一条破 COLD。此时别碰关注钩子/主页装修——钩子作用在小基数上没意义。
2. **已出池（均观看 ≥ COLD）** → 问题从"出不出池"变成"**能不能复制**"。再看转化：
   - 主页转涨粉**健康** + 观看转主页**低** → 瓶颈在"没人看见"不在"留不住"，**别在转化上耗**，唯一杠杆是把观看做大（复制标杆）。
   - 主页转涨粉**低** → 人来了没留住，置顶标杆 + bio 讲清"关注能得到什么" + 封面风格统一。
3. 末尾按样本量（快照数/笔记数）追加**置信度**提示。

## 建议引擎（围绕"复制标杆"，每条挂账号真实数字）

`winner` = 观看最高那条笔记（动态识别）。建议永远以它为锚，实验化输出（动作 + 目标 + 怎么验证）。

| 触发条件 | 优先级 | 给的建议（要点） |
|---|---|---|
| 有笔记（永远） | P0 | **复制标杆**：拆 `winner` 的选题/标题/封面公式，下一条复制，验证能否破 `winner观看×0.8`；还在涨就先置顶吃长尾 |
| 转化健康+曝光低 | P1 | **把力气下对环节**：别在关注钩子上耗，这阶段唯一杠杆是把观看做大 |
| 转化低 | P1 | **主页留人**：置顶标杆、bio 讲清关注收益、封面统一 |
| 永远 | 底线 | 不买量不互关刷粉——僵尸粉伤权重毁转化（压成一行） |

> 已**删除**旧引擎的通用箴言（"封面用大字+数字反差""结尾加钩子""发布后引导评论""好内容返工"）——任何账号都适用＝零增量，稀释价值。结论区的「本周一件事」是 P0 的口语化提炼。

## 闭环验证（复盘历史.json）

每跑一次 analyze 追加一条快照记录（同一快照日重跑则替换，保证每日一条）：

```json
{"最新日期","生成时间","粉丝","观看","出池数","笔记数","标杆标题","标杆观看","本周一件事","笔记keys"}
```

下次跑时取"最新日期不同的最近一条"作上期，自动算：① 粉丝/观看/出池数的 ↑↓ 变化；② 对比 `笔记keys` 判断有没有发新内容（执行检测），新内容观看 vs 上期标杆×0.6 判定向好。首期无对比则报告显示占位提示。

## 守内容真实（与品牌一致）

建议只围绕**内容/封面/标题/节奏/主页装修**这些真实可做的事，不走"抢眼球/标题党/刷量"的捷径。
口径参考工厂 memory `genuine-reading-over-clickbait`：先把内容做对，再谈传播。
