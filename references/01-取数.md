# 01 · 取数层

## 数据来源：opencli 小红书创作者后台

前置：opencli + Chrome 登录态 + opencli 扩展已连。`opencli doctor` 看到 `[OK] Extension` 才能拉。
`pull.py` 用到这三个命令（都只读公开后台）：

| 命令 | 取什么 | 落到哪张 CSV |
|---|---|---|
| `opencli xiaohongshu creator-profile` | 粉丝数 / 获赞藏 / 等级进度 | `小红书粉丝日志.csv` |
| `opencli xiaohongshu creator-stats` | 近 7 日：观看/主页访问/赞/藏/评/分享/涨粉 | `小红书概览日志.csv` |
| `opencli xiaohongshu creator-notes` (json) | 逐篇：**全部已发布笔记** id/标题/观看/赞/藏/评 | `小红书数据日志.csv` |
| `opencli xiaohongshu creator-note-detail <id>` | 每篇：曝光/封面CTR/平均观看时长/**观看来源**/**观众画像** | `小红书单篇详情.csv` |

> ⚠️ 用 `creator-notes`（json）不用 `creator-notes-summary`：summary 只返回最近 3–4 篇（会让报告的出池率虚高、标杆失真），creator-notes 返回**全部**笔记且带 note_id（详情/限流检测必需）。pull 对每篇再调一次 `creator-note-detail`（本地创作者后台，**无需主站登录**）。

其它可用命令：`feed`（首页推荐流，需主站登录，选题雷达用）、`search`（被登录墙挡，慎用）、`comments`（评论/舆情）、`note <signed-url>`（他人笔记详情，url 须带 xsec_token）。

## CSV schema（时间序列，每次拉取追加一行/组）

```
小红书粉丝日志.csv : 拉取日期, 粉丝数, 获赞藏
小红书概览日志.csv : 拉取日期, 观看, 主页访问, 赞, 藏, 评, 分享, 涨粉   # 近7日总量快照
小红书每日趋势.csv : 日期, 观看, 主页访问, 涨粉                        # 来自 creator-stats 的 trend 数组(近7天每日值)，趋势图开箱即 7 点
小红书数据日志.csv : 拉取日期, 标题, 发布时间, 观看, 赞, 藏, 评, 完播   # 全部笔记/每次快照(完播=平均观看时长,取自详情)
小红书单篇详情.csv : 拉取日期, note_id, 标题, 曝光, 观看, 封面点击率, 平均观看时长, 涨粉, 视频推荐, 首页推荐, 关注页面, 搜索, 个人主页, 其他来源, 推荐占比, 男, 女, 主年龄, 主年龄占比, top城市, top兴趣
                   # 低流量笔记无来源/画像数据则该篇跳过(不写空行)；限流检测/流量结构/受众画像/CTR 都吃这张
复盘历史.json      : 每期结论+关键指标快照，闭环验证用（每个快照日一条）
拉取日志.txt       : 每次拉取的成功/失败记录（拉不到时只写这里，不写脏数据）
```

> 编码 `utf-8-sig`（带 BOM，Excel 直接打开不乱码）。`analyze.py` 读时用 DictReader，加列向后兼容。

## 每日自动取数（macOS launchd）

云端 /schedule 跑不了——取数依赖**本机 Chrome 登录态**。用 launchd 本地定时：

```xml
<!-- ~/Library/LaunchAgents/com.hekouwang.xhs-pull.plist 关键字段 -->
ProgramArguments: [/opt/homebrew/bin/python3, <skill>/scripts/pull.py, --data-dir, <绝对路径>/渠道数据分析师]
EnvironmentVariables PATH: $HOME/.local/share/fnm/aliases/default/bin:/opt/homebrew/bin:/usr/bin:/bin
# ↑ 写进 plist 时 $HOME 要换成展开后的绝对家目录（launchd 不展开 ~/$HOME）
WorkingDirectory: <项目根>
StartCalendarInterval: {Hour 21, Minute 30}
```

```bash
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.hekouwang.xhs-pull.plist  # 装/重载先 bootout 同名
launchctl print gui/$(id -u)/com.hekouwang.xhs-pull | grep -i state                    # 验证
```

⚠️ **铁律**：launchd 的 PATH 极简，`opencli`/`node` **必须用 fnm 稳定路径**
`~/.local/share/fnm/aliases/default/bin`，**别用带会话号的 `fnm_multishells/<num>_<ts>` 临时路径**（重启即失效）。
⚠️ **物理限制**：到点时须 Mac 醒着 + Chrome 开着，否则当次跳过（脚本只记失败日志）。

## 扩展：加渠道 / 加账号

- **加账号**：每个账号给独立 `--data-dir`（如 `客户A/数据/`），analyze/build_report 同样传 `--data-dir`。
- **加渠道**：在 `pull.py` 加对应平台的拉取分支 + 新 CSV schema；agent-reach/opencli 已支持的：B站（`creator-*` 视频后台）、其它走 `agent-reach doctor --json` 看后端。抖音/视频号/公众号/头条无现成只读后端，可能需手工录入或网页自动化（opencli browser）。
- 加渠道后同步更新 `analyze.py`（指标可能不同）与 `03-报告设计.md`（报告分区）。
