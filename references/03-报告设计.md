# 03 · 报告设计（V2 米白 复盘报告 HTML）

> 生成器 `scripts/build_report.py`：读 `report_data.json` → 输出自托管单文件 HTML。零依赖、自制 SVG 图表。

## 视觉规范（V2 米白，与 content-factory 一致）

- 底色 `#faf9f5`，主色黏土 `#c15f3c`，辅 石板蓝 `#5c6b7a` / 赭石 `#a07a3c`，风险砖红 `#c0392b`。
- 字体：中文思源黑体（Google Fonts）+ 拉丁/数字 Anthropic Sans/Mono（取 content-factory 内置 woff2）。
- 卡片：白底 + 1px 细边 + 柔和投影（亮底靠投影出层次）。**禁左侧彩色竖条**（工厂铁律）。
- 数字一律 mono 字体，强化"数据感"。

## 报告结构（自上而下）

1. **报头 sysbar**：账号 · 平台 · "渠道数据分析"。
2. **大标题**：渠道数据复盘报告 + 中立声明副标 + 数据区间/生成时间。
3. **KPI 四卡**：粉丝数(+环比) · 近7日观看 · 近7日涨粉(每千观看涨粉) · 获赞藏(距500等级)。
4. **转化漏斗 + 双仪表盘**：ECharts 漏斗(观看→主页→涨粉) + 两个 gauge(两道转化率，按健康区间红/黄/绿着色)。
5. **涨粉目标测算**：到 500 / 1000 粉各需多少累计观看、多少篇出池笔记 + **杠杆测算**(把转化率提到健康线，所需笔记数对比)。这是客户最想要的"还要多久/还要多少"。
6. **指标基准对照**：你的值 vs 健康线 + 评级 badge。
7. **趋势**：ECharts 双轴折线(粉丝 + 近7日观看)；<2 天快照显示"数据积累中"占位。
8. **笔记红黑榜 · 逐篇判决**：ECharts 横向条形(出池绿/未出池红 + 出池线 300 markLine) + 表格带**判决列**(置顶/保持/换封面返工/封面+选题重做)。
9. **诊断**：项目符号列表，自上而下定位瓶颈。
10. **本周行动清单**：可勾选待办(从判决+建议提炼)。
11. **行动建议**：分优先级卡（P0 红 / P1 黏土 / P2 石板 / 底线 灰）。
12. **落款 footer**：品牌 + 数据来源说明条（合规，勿删）。

## 图表（ECharts，内联自托管）

- 用 ECharts（`assets/echarts.min.js`，build 时**内联进 HTML** → 单文件、离线可截图）。
- **漏斗**：funnel series，`minSize:'26%'` 保证涨粉小段可见；标签显示真实数值。
- **仪表盘**：gauge，`axisLine` 三段色按 [偏低/及格/健康] 阈值着色，指针指当前值。
- **趋势**：双轴 line(粉丝左轴/观看右轴)，`smooth+areaStyle`；单点时 graphic 文本提示。
- **条形**：横向 bar，按出池布尔着色(绿/红) + `markLine` 画出池线 300。
- 全局 `animation:false`（截图取最终态）；字体 `Noto Sans SC`，配色用 V2 token 值。
- ⚠️ 截图须 `--virtual-time-budget=8000`（等 canvas 渲染）；报告约 1MB(含 echarts)。

## ECharts 防踩坑（实测，改图必查 ⭐）

逐条都是真出过的问题，改 `build_report.py` 图表前对照一遍：

1. **仪表盘指针压住数字**（gauge）：别把 `detail`(数字) 放圆心——指针会穿过它。做法：数字下移到弧下方
   `detail.offsetCenter:[0,'62%']`、标题再低 `title.offsetCenter:[0,'92%']`、指针缩短 `pointer.length:'46%'`、
   圆心加 `anchor`、`center:['50%','52%']` 上移留出下方空间、容器高 ≥172px。
2. **横向条形 Y 轴长标题被从左裁**（bar）：①`grid.left` 要够大（中文长标题给 ~170）；
   ②`yAxis.axisLabel` 必须设 `width:152, overflow:'truncate', align:'right'`，让 ECharts 自己右对齐截断；
   ③**别在 Python 端硬截断标题**（`[:14]+…`）——交给 ECharts 的 `overflow:'truncate'`，否则左侧仍可能被网格裁。
3. **markLine 标签顶部被裁**：垂直 markLine(如出池线 300) 的标签默认在顶端，会被 `grid.top` 裁掉。
   做法：`grid.top` 留 ≥34、`markLine ... label.position:'end'`。
4. **截图空白/动画态**：所有图 `animation:false`；headless 截图 `--virtual-time-budget=8000` 等 canvas 画完。
5. **单点趋势**：快照 <2 天时折线只有一个点，用 `graphic` 文本提示"数据积累中"，别让图看着像坏了。
6. **funnel 小段不可见**：涨粉 << 观看，漏斗末段会细到看不见——设 `minSize:'26%'` 保底，标签显真实值。

## 自用版 vs 客户版（落款差异）

- **自用版（v1 默认）**：footer = "会勇禾口王 · 数据复盘" + 来源说明。纯复盘，无销售。
- **客户版（扩展）**：把 footer 落款换成**服务方品牌 + 联系方式 + 服务 CTA**（"想要每周这样一份？→ 加微信"），并在报告头部加客户账号名。实现方式：给 `build_report.py` 加 `--mode client --brand "<服务方>" --cta "<联系方式>"`，模板里条件渲染。**做客户版前先在 04-服务化.md 对齐定价与交付。**

## 交付格式

- **HTML**：直接发，对方浏览器打开（自托管字体，离线也能看大部分）。
- **PDF**：Chrome 打印成 PDF，或 `assets/fullpage.js`（content-factory）出长图。
- **长图**：截整页 PNG 发小红书/朋友圈当案例物料。

## 截图命令（验证/出图）

```bash
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" --headless=new \
  --allow-file-access-from-files --disable-gpu --no-sandbox --window-size=1180,2600 \
  --hide-scrollbars --screenshot=preview.png --virtual-time-budget=8000 \
  "file://<绝对路径>/渠道数据复盘报告.html"
```
（带 `--allow-file-access-from-files` 才能加载本地 woff2；长报告用 fullpage.js 出完整长图。）
