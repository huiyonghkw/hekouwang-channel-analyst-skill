# 会勇禾口王 · 渠道数据分析师

> 把账号的公开后台数据，做成一份能看、能交付、能指导动作的复盘报告。
> 链路：**取数（pull）→ 分析（analyze）→ 报告（build_report）→ 建议**。
> v1 覆盖小红书；结构按多渠道/多账号预留（扩展见 `references/01-取数.md`）。

## 0. 定位与边界

- **分析师 ≠ 仪表盘**：用户不缺数字（后台一打开全是），缺的是「判断」。价值在把数字翻译成"这周该赌哪边、有多大把握"，并帮用户**少做无用功**。报告一律**判断在前、数据降为证据**。
- **自用优先**：先把作者自己的账号跑顺、当样板案例；客户版（带服务落款/CTA）见 `references/04-服务化.md`。
- **交付物**：`渠道数据复盘报告.html`（V2 米白，自托管单文件，可截图/转 PDF/转长图）。
- **不是什么**：不是发布工具（不替用户发内容）、不是刷量工具、不预测涨多少粉、不是"把后台数字画成图"的仪表盘。

## 0.5 ⚠️ 合规底线（每次产出前自检，先看再做）

1. **不刷量、不互关刷粉、不买粉**——僵尸粉伤权重、毁私域转化，建议里绝不出现。
2. **不承诺具体涨粉数字 / 不打"包涨"包票**——只做基于历史数据的中立复盘，结论写"趋势/概率/建议"，不写"保证"。
3. **数据只来自账号自己的公开创作者后台**（用户已登录授权），不抓他人隐私、不爬非授权数据。
4. 报告底部固定保留来源说明条（模板已内置），别删。
5. 命中以上任意越界 → 停下说明，不直接产出。

## 1. 三步流水线（命令）

脚本在本 skill 的 `scripts/`。数据默认写到**当前工作目录下的 `渠道数据分析师/`**（可用 `--data-dir` 改；多账号给各自目录）。前置：opencli + Chrome 登录态 + 扩展已连（`opencli doctor` 看 `[OK] Extension`）。

```bash
# ① 取数：拉粉丝/漏斗/逐篇，追加进 CSV 时间序列（拉不到只记日志、不写脏数据）
python3 <skill>/scripts/pull.py --data-dir 渠道数据分析师
# ② 分析：CSV → 指标/趋势/诊断/建议 → report_data.json
python3 <skill>/scripts/analyze.py --data-dir 渠道数据分析师
# ③ 报告：JSON → V2 米白 渠道数据复盘报告.html
python3 <skill>/scripts/build_report.py --data-dir 渠道数据分析师
```

> `<skill>` = 本 skill 加载时声明的 Base directory。报告字体默认引用 `hekouwang-content-factory` 内置的 Anthropic 字体；换机时用 `build_report.py --font-dir` 指定。
> **每日自动取数**：用 macOS launchd 定时跑 `pull.py`（配方见 `references/01-取数.md`）。云端 /schedule 跑不了——取数依赖本地 Chrome 登录态。

## 2. 分析方法论（判断优先 · 细节见 `references/02-分析框架.md`）

**铁律：先给判断，再给数据。** 价值阶梯：描述 < 诊断 < 预测 < 指示——钱在上三层；只"画图展示"是仪表盘，零增量。报告固定结构（`build_report.py` 已实现）：

1. **结论**（英雄区）：账号状态一句话 + **本周唯一一件事** + 止损「别做什么」+ 置信度。
2. **上期复盘**：闭环验证——上次让你做什么 → 数据怎么动 → 有没有执行（数据存 `复盘历史.json`）。
3. **为什么(诊断) + 怎么做(建议)**：判断层。
4. **支撑数据**：KPI / 漏斗 / 趋势 / 红黑榜，降级为证据放在分隔线之后。

七个价值点（按被低估程度，细节见 `references/02`）：

- **闭环验证**：上期建议到底有没有用——analyst 与"报告生成器"的分水岭，没闭环的建议都是放空炮。
- **止损/聚焦**：结论只给一件事并显式说"别做什么"，负向建议比正向更省用户时间。
- **复制标杆**：建议永远围绕观看最高那条（动态识别）展开，把"偶然爆"变"可复制"，每条挂账号真实数字。
- **归因**：给"为什么"不只给"是什么"。**基准**：自己过去(环比)/同赛道(对标)/目标三把尺子。**预警**：趋势方向。**置信度诚实**：样本薄就标"方向性判断"，不假装确定。
- 底层仍是**涨粉漏斗**（观看→主页→涨粉两道转化率定位瓶颈）+ **出池诊断**（单篇观看<阈值默认300≈卡冷启动池）。
- **单篇深挖**（`creator-note-detail`，本地后台无需主站登录）：**限流检测**（推荐占比<40%=没进/被限推荐池）·**流量结构**（推荐/搜索/关注占比）·**受众画像**（性别/年龄/城市/兴趣，验人设契合）·**封面CTR/完播**（真实数字，判断"卡点是不是封面"）。取数走 `creator-notes`(全部笔记)，别用 summary(只 3 篇会让出池率虚高)。

## 生态依赖（与哪些 skill / 工具协作）

性质＝**软依赖**：核心三件套（pull/analyze/build_report）能独立跑；下面是协作关系，不是硬绑定。

| 依赖 | 类型 | 用在哪 | 缺了会怎样 |
|---|---|---|---|
| OpenCLI / `agent-reach` | 工具（必需取数） | 复用 Chrome 登录态拉创作者后台 | 取不到数；但已有 CSV 仍可分析+出报告 |
| `hekouwang-content-factory` | 软依赖 | 报告 V2 米白字体/视觉取自它；**封面返工/小红书图**也走它 | 字体回退——`build_report.py --font-dir` 换系统字体即可 |
| `hekouwang-claude-skill-doctor-skill` | 质量 | 本 skill 自身体检（现 100/100） | 不影响运行 |
| Apache ECharts | 内置 | 报告图表（已内联进 `assets/`） | 已随仓库分发，无需联网 |
